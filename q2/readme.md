## Data

### Automatic (LibriSpeech)
No manual download needed. The dataset will be downloaded automatically to
`./data/` on first run via `torchaudio.datasets.LIBRISPEECH`.

### Optional: MUSAN noise (recommended for realistic env. mismatch)
```bash
wget http://www.openslr.org/resources/17/musan.tar.gz
tar xzf musan.tar.gz -C ./data/
```
Then set `noise_dir: ./data/musan/noise` in all configs.

## Training

### Step 1 — Baseline
```bash
python train.py --config configs/baseline.yaml
# Saves: exps/baseline/ckpt_epoch*.pt
```

### Step 2 — Paper method (Disentangler)
```bash
python train.py --config configs/disentangler.yaml
# Saves: exps/disentangler/ckpt_epoch*.pt
```

### Step 3 — Proposed improvement (InfoNCE)
```bash
python train.py --config configs/improved.yaml
# Saves: exps/improved/ckpt_epoch*.pt
```

Training takes ~2–4 hours per run on a single T4 GPU.
Use `--seed` to fix randomness (default: 42).

## Evaluation
```bash
python eval.py \
  --config configs/disentangler.yaml \
  --checkpoints \
    exps/baseline/ckpt_epoch050.pt \
    exps/disentangler/ckpt_epoch050.pt \
    exps/improved/ckpt_epoch050.pt \
  --names baseline disentangler improved \
  --results_dir results/
```

If no `--trial_list` is provided, the script automatically builds synthetic
trials from LibriSpeech test-clean (genuine pairs from same speaker,
impostor pairs across speakers).

Outputs:
- `results/tables/results.csv`
- `results/plots/det_all_systems.png`
- `results/plots/score_dist_*.png`
- `results/plots/tsne_*.png`

## Justification of reduced reproduction

| Aspect             | Paper                  | This reproduction       |
|--------------------|------------------------|-------------------------|
| Backbone           | ResNetSE34V2/ECAPA-TDNN| Lightweight TDNN        |
| Training data      | VoxCeleb2 dev          | LibriSpeech train-100   |
| Session labels     | YouTube video IDs      | LibriSpeech chapter IDs |
| Eval sets          | Vox1-O, VoxSRC22/23    | LibriSpeech test-clean  |
| Environment mismatch | Real recording conditions | MUSAN augmentation  |
| Disentangler       | Identical architecture | ✓ faithful reproduction |
| Loss functions     | All 5 losses           | ✓ all 5 implemented     |
| Code swapping      | ✓                      | ✓ implemented           |
| GRL                | ✓                      | ✓ implemented           |

The core algorithmic contribution (AE disentangler + 5-loss objective +
code swapping + GRL) is faithfully reproduced. Compute constraints motivate
the lighter backbone and smaller dataset.

## Proposed improvement

**InfoNCE contrastive loss on speaker subspace** (in `improved.yaml`).

**Motivation:** The paper's L_spk is a classification loss that requires
fixed speaker IDs and does not directly enforce within-speaker invariance
across environments. InfoNCE treats (z_spk from env_1, z_spk from env_2)
of the same speaker as a positive pair, directly pulling them together
while pushing apart different speakers — regardless of speaker labels.
This operationalises environment invariance at the representation level.

See Section 7 of `review.pdf` for full theoretical justification.
